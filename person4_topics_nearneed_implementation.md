## 1. Real-Time Messaging System

### Brief Explanation

The real-time messaging system enables **instant communication** between Seekers and Providers. Messages are stored in Firestore with real-time listeners, supporting text, images, and voice messages with delivery/read status tracking.

### Implementation Flow

```
User types message
    ↓
Message validated (length, empty checks)
    ↓
Create ChatMessage object with timestamp
    ↓
Upload to Firestore: /chats/{chatId}/messages/{messageId}
    ↓
Real-time listener on receiver side picks up message
    ↓
Display in RecyclerView with read receipts
    ↓
Mark as read when user views it
```

### Java Code Implementation

#### **ChatMessage.java** - Data Model

```java
package com.example.nearneed;

/**
 * Model representing a chat message.
 * Firestore path: messages/{chatId}/messages/{messageId}
 *
 * This class represents individual messages in conversations.
 * Fields marked as "transient" are UI-only and NOT stored in Firestore.
 */
public class ChatMessage {
    public String messageId;           // Unique identifier for message
    public String senderId;            // UID of the sender
    public String messageText;         // Actual message content
    public Long timestamp;             // When message was sent (milliseconds)

    // Media fields for rich messaging
    public String audioPath;           // Path to recorded voice message
    public String imageUri;            // URI of attached image

    // UI-only state fields (transient, NOT stored in Firebase)
    public transient boolean isOutgoing;   // Is this from current user?
    public transient boolean isVoice;      // Is this a voice message?
    public transient boolean isPlaying;    // Is audio currently playing?
    public transient int progress;         // Audio playback progress (0-100)
    public transient int durationSecs;     // Duration of voice message in seconds

    // Default constructor (required by Firestore)
    public ChatMessage() {}

    // Constructor for text message
    public ChatMessage(String senderId, String messageText) {
        this.senderId = senderId;
        this.messageText = messageText;
        this.timestamp = System.currentTimeMillis();  // Server timestamp
    }

    // Constructor for media messages
    public ChatMessage(String text, boolean isVoice, boolean isOutgoing) {
        this.messageText = text;
        this.isVoice = isVoice;
        this.isOutgoing = isOutgoing;
        this.timestamp = System.currentTimeMillis();
    }
}
```

**Key Functions Explained:**

- `messageId`: Unique identifier generated by Firestore automatically
- `timestamp`: Milliseconds since epoch; used for sorting and read receipts
- `transient` keyword: Fields marked transient are NOT serialized to/from Firestore
- `isOutgoing`: Determines if message appears on right (true) or left (false) in UI

#### **ChatActivity.java** - Messaging Interface

```java
package com.example.nearneed;

import android.os.Bundle;
import android.widget.EditText;
import android.widget.ImageView;
import android.widget.RecyclerView;
import androidx.appcompat.app.AppCompatActivity;
import androidx.recyclerview.widget.LinearLayoutManager;
import com.google.firebase.auth.FirebaseAuth;
import java.util.ArrayList;
import java.util.List;

public class ChatActivity extends AppCompatActivity {

    private RecyclerView rvMessages;           // List of messages
    private ChatAdapter adapter;               // Adapter to display messages
    private List<ChatMessage> messageList = new ArrayList<>();
    private EditText etMessageInput;           // Input field for typing
    private ImageView btnSend, btnAdd, btnBack;
    private String chatId;                     // Current conversation ID
    private String otherUserId;               // ID of the other participant
    private String currentUserId;             // Current user's ID

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_chat);

        // Initialize UI components
        rvMessages = findViewById(R.id.rvMessages);
        etMessageInput = findViewById(R.id.etMessageInput);
        btnSend = findViewById(R.id.btnSend);
        btnAdd = findViewById(R.id.btnAdd);
        btnBack = findViewById(R.id.btnBack);

        // Setup RecyclerView for message list
        rvMessages.setLayoutManager(new LinearLayoutManager(this));
        adapter = new ChatAdapter(messageList);
        rvMessages.setAdapter(adapter);

        // Get conversation details from Intent
        chatId = getIntent().getStringExtra("CHAT_ID");
        otherUserId = getIntent().getStringExtra("CHAT_USER_ID");
        currentUserId = FirebaseAuth.getInstance().getUid();

        // Setup button listeners
        setupButtonListeners();

        // Load messages in real-time
        loadMessagesRealtime();
    }

    /**
     * Setup click listeners for UI buttons
     */
    private void setupButtonListeners() {
        // Back button - finish activity
        btnBack.setOnClickListener(v -> finish());

        // Send button - send text message
        btnSend.setOnClickListener(v -> sendTextMessage());

        // Add button - show options for media (image, voice, etc)
        btnAdd.setOnClickListener(v -> showMediaOptions());
    }

    /**
     * Send a text message to Firestore
     *
     * Process:
     * 1. Get text from input field
     * 2. Validate it's not empty
     * 3. Create ChatMessage object
     * 4. Upload to Firestore /chats/{chatId}/messages/{messageId}
     * 5. Clear input field
     * 6. Scroll to latest message
     */
    private void sendTextMessage() {
        String text = etMessageInput.getText().toString().trim();

        if (text.isEmpty()) {
            return; // Don't send empty messages
        }

        // Create message object
        ChatMessage message = new ChatMessage(currentUserId, text);
        message.isOutgoing = true; // Mark as from current user

        // Upload to Firestore
        com.google.firebase.firestore.FirebaseFirestore db =
            com.google.firebase.firestore.FirebaseFirestore.getInstance();

        db.collection("chats")
            .document(chatId)
            .collection("messages")
            .add(message)
            .addOnSuccessListener(docRef -> {
                message.messageId = docRef.getId(); // Store generated ID
                messageList.add(message);
                adapter.notifyItemInserted(messageList.size() - 1);
                rvMessages.smoothScrollToPosition(messageList.size() - 1);
                etMessageInput.setText(""); // Clear input
            })
            .addOnFailureListener(e -> {
                android.widget.Toast.makeText(this,
                    "Failed to send: " + e.getMessage(),
                    android.widget.Toast.LENGTH_SHORT).show();
            });
    }

    /**
     * Load messages in real-time from Firestore
     * This uses a Firestore listener which:
     * - Fetches initial messages
     * - Listens for new messages added
     * - Updates UI automatically when messages arrive
     */
    private void loadMessagesRealtime() {
        com.google.firebase.firestore.FirebaseFirestore db =
            com.google.firebase.firestore.FirebaseFirestore.getInstance();

        db.collection("chats")
            .document(chatId)
            .collection("messages")
            .orderBy("timestamp", com.google.firebase.firestore.Query.Direction.ASCENDING)
            .addSnapshotListener((querySnapshot, error) -> {
                if (error != null) {
                    android.widget.Toast.makeText(this,
                        "Error loading messages",
                        android.widget.Toast.LENGTH_SHORT).show();
                    return;
                }

                messageList.clear();

                // Convert each Firestore document to ChatMessage
                for (com.google.firebase.firestore.DocumentSnapshot doc :
                        querySnapshot.getDocuments()) {
                    ChatMessage msg = doc.toObject(ChatMessage.class);
                    if (msg != null) {
                        msg.messageId = doc.getId();
                        // Mark as outgoing if sent by current user
                        msg.isOutgoing = currentUserId.equals(msg.senderId);
                        messageList.add(msg);
                    }
                }

                adapter.notifyDataSetChanged(); // Refresh UI
                // Scroll to latest message
                if (!messageList.isEmpty()) {
                    rvMessages.smoothScrollToPosition(messageList.size() - 1);
                }
            });
    }

    private void showMediaOptions() {
        // Implementation for image/voice selection
    }
}
```

**Key Functions Explained:**

- `sendTextMessage()`: Validates input, creates message, uploads to Firestore, updates UI
- `loadMessagesRealtime()`: Attaches Firestore listener for real-time updates
- `addSnapshotListener()`: Firebase method that listens for changes and calls callback automatically
- `orderBy("timestamp")`: Ensures messages are sorted chronologically

### Firebase Structure

```
Firestore:
├── chats/
│   └── {chatId}  (e.g., "user1_user2")
│       ├── participantIds: ["user1", "user2"]
│       ├── lastMessage: "Hey, how are you?"
│       ├── lastMessageTime: timestamp
│       └── messages/ (subcollection)
│           └── {messageId} (auto-generated)
│               ├── senderId: "user1"
│               ├── messageText: "Hey, how are you?"
│               ├── timestamp: 1682774400000
│               ├── imageUri: "gs://..." (if image)
│               └── audioPath: "/path/to/audio" (if voice)
```

## 2. Booking Management Interface

### Brief Explanation

Booking management provides a **centralized hub** for users to view, manage, and track all bookings. Different interfaces exist for Seekers (requesting services) and Providers (offering services), showing status, pricing, and action buttons.

### Implementation Flow

```
User opens BookingsActivity
    ↓
Load bookings from Firestore based on role
    ↓
Display in RecyclerView (Seeker/Provider specific view)
    ↓
Filter/Sort by status, date, price
    ↓
User can:
  - View details
  - Message other party
  - Cancel booking
  - Complete booking
  - Leave review
```

### Java Code Implementation

#### **Booking.java** - Data Model

```java
package com.example.nearneed;

/**
 * Booking data model representing a service request agreement
 * Firestore path: bookings/{bookingId}
 *
 * A booking connects a Seeker (requesting service) with a Provider
 * (offering service) for a specific Post.
 */
public class Booking {
    public String bookingId;          // Unique booking identifier
    public String postId;             // Which post/service is this for?
    public String seekerId;           // Who requested the service?
    public String providerId;         // Who will provide the service?
    public String applicationId;      // Link to job application if any

    // Naming & type info (cached for display)
    public String postTitle;
    public String postType;           // "GIG" or "COMMUNITY"
    public String seekerName;
    public String providerName;

    // Timing
    public long createdAt;            // When booking was created
    public long scheduledDate;        // When service is scheduled

    // Payment info
    public Double amount;             // Service cost
    public String status;             // "upcoming", "ongoing", "completed", "cancelled"

    // Default constructor (required by Firestore)
    public Booking() {}

    // Constructor with IDs
    public Booking(String postId, String seekerId, String providerId) {
        this.postId = postId;
        this.seekerId = seekerId;
        this.providerId = providerId;
        this.createdAt = System.currentTimeMillis();
    }

    // Getters
    public String getStatus() { return status; }
    public Double getAmount() { return amount; }
    public String getProviderName() { return providerName; }
}
```

#### **BookingRepository.java** - Data Access Layer

```java
package com.example.nearneed;

import android.content.Context;
import com.google.firebase.auth.FirebaseAuth;
import com.google.firebase.firestore.FirebaseFirestore;
import com.google.firebase.firestore.ListenerRegistration;
import java.util.ArrayList;
import java.util.List;

/**
 * Repository for managing Booking data with Firestore
 * Handles CRUD operations and real-time listeners
 *
 * Separation of concerns: Repository handles data layer,
 * ViewModel handles UI logic, Activity handles UI rendering
 */
public class BookingRepository {

    private static final String BOOKINGS_COLLECTION = "bookings";

    // Callback interfaces for async operations
    public interface BookingListener {
        void onBookingsLoaded(List<Booking> bookings);
        void onError(Exception e);
    }

    public interface SaveCallback {
        void onSuccess(String bookingId);
        void onFailure(Exception e);
    }

    /**
     * Create a new booking (Simple version with just IDs and amount)
     *
     * @param postId        Which post/service
     * @param seekerId      Who is requesting
     * @param providerId    Who is providing
     * @param amount        How much to pay
     * @param callback      Result callback
     */
    public static void createBooking(String postId, String seekerId,
                                     String providerId, Double amount,
                                     SaveCallback callback) {
        createBooking(postId, null, null, seekerId, providerId, null, amount, null, callback);
    }

    /**
     * Create a new booking (Full version with all details)
     *
     * Steps:
     * 1. Fetch seeker name from users collection
     * 2. Fetch provider name from users collection
     * 3. Create Booking object with all info
     * 4. Upload to Firestore
     * 5. Return generated booking ID
     */
    public static void createBooking(String postId, String postTitle, String postType,
                                     String seekerId, String providerId, String applicationId,
                                     Double amount, Long scheduledDate, SaveCallback callback) {
        if (postId == null || callback == null) return;

        FirebaseFirestore db = FirebaseFirestore.getInstance();
        final String[] seekerName = {""};        // Will hold seeker's name
        final String[] providerName = {""};      // Will hold provider's name
        final int[] fetchCount = {0};            // Track how many fetches done

        // Callback that runs when both names are fetched
        Runnable saveBooking = () -> {
            fetchCount[0]++;
            if (fetchCount[0] < 2) return;  // Wait until both names fetched

            // Create booking with all details
            Booking booking = new Booking(postId, seekerId, providerId);
            booking.postTitle = postTitle;
            booking.postType = postType;
            booking.applicationId = applicationId;
            booking.createdAt = System.currentTimeMillis();
            booking.scheduledDate = scheduledDate != null ? scheduledDate : System.currentTimeMillis();
            booking.seekerName = seekerName[0];
            booking.providerName = providerName[0];
            booking.amount = amount;
            booking.status = "upcoming";

            // Upload to Firestore
            db.collection(BOOKINGS_COLLECTION)
                    .add(booking)
                    .addOnSuccessListener(docRef -> callback.onSuccess(docRef.getId()))
                    .addOnFailureListener(e -> callback.onFailure(e));
        };

        // Fetch seeker's name
        if (seekerId != null && !seekerId.isEmpty()) {
            db.collection("users").document(seekerId).get()
                    .addOnSuccessListener(doc -> {
                        if (doc != null && doc.exists()) {
                            String name = doc.getString("name");
                            if (name == null || name.isEmpty()) {
                                name = doc.getString("fullName");
                            }
                            if (name != null && !name.isEmpty()) {
                                seekerName[0] = name;
                            }
                        }
                        saveBooking.run();  // Check if ready to save
                    })
                    .addOnFailureListener(e -> saveBooking.run());
        } else {
            fetchCount[0]++;
            saveBooking.run();
        }

        // Fetch provider's name
        if (providerId != null && !providerId.isEmpty()) {
            db.collection("users").document(providerId).get()
                    .addOnSuccessListener(doc -> {
                        if (doc != null && doc.exists()) {
                            String name = doc.getString("name");
                            if (name == null || name.isEmpty()) {
                                name = doc.getString("fullName");
                            }
                            if (name != null && !name.isEmpty()) {
                                providerName[0] = name;
                            }
                        }
                        saveBooking.run();  // Check if ready to save
                    })
                    .addOnFailureListener(e -> saveBooking.run());
        } else {
            fetchCount[0]++;
            saveBooking.run();
        }
    }

    /**
     * Observe user's bookings in real-time
     * Listens to Firestore for changes and calls callback automatically
     *
     * @param context    Android context
     * @param listener   Callback for when bookings load or error
     * @return           ListenerRegistration to stop listening later
     */
    public static ListenerRegistration observeUserBookings(Context context,
                                                           BookingListener listener) {
        String userId = FirebaseAuth.getInstance().getUid();
        if (userId == null) {
            listener.onError(new Exception("Not logged in"));
            return null;
        }

        FirebaseFirestore db = FirebaseFirestore.getInstance();

        // Query bookings where user is seeker OR provider
        return db.collection(BOOKINGS_COLLECTION)
                .whereEqualTo("seekerId", userId)
                .addSnapshotListener((snapshot, error) -> {
                    if (error != null) {
                        listener.onError(error);
                        return;
                    }

                    List<Booking> bookings = new ArrayList<>();
                    if (snapshot != null) {
                        // Convert Firestore documents to Booking objects
                        for (com.google.firebase.firestore.DocumentSnapshot doc : snapshot.getDocuments()) {
                            Booking booking = doc.toObject(Booking.class);
                            if (booking != null) {
                                booking.bookingId = doc.getId();
                                bookings.add(booking);
                            }
                        }
                    }

                    listener.onBookingsLoaded(bookings);
                });
    }

    /**
     * Observe bookings for a specific post
     * Used when provider wants to see all requests for their posted service
     */
    public static ListenerRegistration observeBookingsForPost(String postId,
                                                              BookingListener listener) {
        if (postId == null) {
            listener.onError(new Exception("Invalid post ID"));
            return null;
        }

        FirebaseFirestore db = FirebaseFirestore.getInstance();

        return db.collection(BOOKINGS_COLLECTION)
                .whereEqualTo("postId", postId)
                .addSnapshotListener((snapshot, error) -> {
                    if (error != null) {
                        listener.onError(error);
                        return;
                    }

                    List<Booking> bookings = new ArrayList<>();
                    if (snapshot != null) {
                        for (com.google.firebase.firestore.DocumentSnapshot doc : snapshot.getDocuments()) {
                            Booking booking = doc.toObject(Booking.class);
                            if (booking != null) {
                                booking.bookingId = doc.getId();
                                bookings.add(booking);
                            }
                        }
                    }

                    listener.onBookingsLoaded(bookings);
                });
    }
}
```

**Key Functions Explained:**

- `createBooking()`: Async operation that fetches names, creates booking, uploads to Firestore
- `observeUserBookings()`: Real-time listener using Firestore's `addSnapshotListener()`
- `whereEqualTo()`: Firestore query filter - only returns matching documents
- `addSnapshotListener()`: Attaches listener that auto-calls callback on any changes

```java
/**
 * ViewModel for Booking data

 * Responsibilities:
 * - Manage LiveData for UI to observe
 * - Handle Firestore listeners lifecycle
 * - Fetch data from Repository
 * - Clean up listeners when ViewModel destroyed
 */
public class BookingViewModel extends AndroidViewModel {

    private MutableLiveData<List<Booking>> userBookings;
    private MutableLiveData<List<Booking>> postBookings;
    private MutableLiveData<String> errorMessage;
    private MutableLiveData<Boolean> isLoading;

    // Listener registrations (for cleanup)
    private ListenerRegistration userBookingsListener;
    private ListenerRegistration postBookingsListener;

    public BookingViewModel(@NonNull Application application) {
        super(application);
        userBookings = new MutableLiveData<>(new ArrayList<>());
        postBookings = new MutableLiveData<>(new ArrayList<>());
        errorMessage = new MutableLiveData<>();
        isLoading = new MutableLiveData<>(false);
    }

    // Getter methods return LiveData for UI to observe

    /**
     * Get user's bookings as LiveData
     * UI observes this: userBookings.observe(viewLifecycleOwner) { ... }
     */
    public LiveData<List<Booking>> getUserBookings() {
        if (userBookings == null) {
            userBookings = new MutableLiveData<>();
            userBookings.setValue(new ArrayList<>());
        }
        return userBookings;
    }

    public LiveData<List<Booking>> getPostBookings() {
        if (postBookings == null) {
            postBookings = new MutableLiveData<>();
            postBookings.setValue(new ArrayList<>());
        }
        return postBookings;
    }

    public LiveData<String> getErrorMessage() {
        if (errorMessage == null) {
            errorMessage = new MutableLiveData<>();
        }
        return errorMessage;
    }

    public LiveData<Boolean> getIsLoading() {
        if (isLoading == null) {
            isLoading = new MutableLiveData<>(false);
        }
        return isLoading;
    }

    /**
     * Start observing user's bookings in real-time
     *
     * Process:
     * 1. Remove old listener if exists
     * 2. Set loading = true
     * 3. Call Repository to attach Firestore listener
     * 4. Update LiveData when data arrives
     * 5. Set loading = false
     */
    public void observeUserBookings() {
        if (userBookingsListener != null) {
            userBookingsListener.remove();  // Stop old listener
        }

        com.google.firebase.auth.FirebaseUser currentUser =
            FirebaseAuth.getInstance().getCurrentUser();

        if (currentUser == null) {
            userBookings.setValue(new ArrayList<>());
            return;
        }

        isLoading.setValue(true);

        // Get listener from repository
        userBookingsListener = BookingRepository.observeUserBookings(
            getApplication(),
            new BookingRepository.BookingListener() {
                @Override
                public void onBookingsLoaded(List<Booking> bookings) {
                    userBookings.setValue(bookings);  // Update LiveData
                    isLoading.setValue(false);
                    errorMessage.setValue(null);
                }

                @Override
                public void onError(Exception e) {
                    errorMessage.setValue("Error loading bookings: " + e.getMessage());
                    isLoading.setValue(false);
                }
            }
        );
    }

    /**
     * Start observing bookings for a specific post
     */
    public void observeBookingsForPost(String postId) {
        if (postBookingsListener != null) {
            postBookingsListener.remove();
        }

        isLoading.setValue(true);

        postBookingsListener = BookingRepository.observeBookingsForPost(
            postId,
            new BookingRepository.BookingListener() {
                @Override
                public void onBookingsLoaded(List<Booking> bookings) {
                    postBookings.setValue(bookings);
                    isLoading.setValue(false);
                    errorMessage.setValue(null);
                }

                @Override
                public void onError(Exception e) {
                    errorMessage.setValue("Error loading post bookings: " + e.getMessage());
                    isLoading.setValue(false);
                }
            }
        );
    }

    /**
     * Clean up listeners when ViewModel is destroyed
     * IMPORTANT: Prevents memory leaks and orphaned listeners
     */
    @Override
    protected void onCleared() {
        super.onCleared();

        if (userBookingsListener != null) {
            userBookingsListener.remove();
        }
        if (postBookingsListener != null) {
            postBookingsListener.remove();
        }
    }
}
```

## 3. Booking State Machine

### Brief Explanation

The **state machine enforces strict workflow** for bookings: `ACTIVE` → `IN_PROGRESS` → `COMPLETED`. It prevents invalid transitions (e.g., cannot complete a cancelled booking) and manages all state-related data.

### Implementation Flow

```
Booking created → Status: ACTIVE
                    ↓
                (User accepts) → Status: IN_PROGRESS
                    ↓
            (User completes) → Status: COMPLETED
                    ↓
            (Can't change from here, but can DISPUTE)

OR from ACTIVE → (Cancel) → Status: CANCELLED
```

### Java Code Implementation

#### **BookingStateManager.java** - In-Memory State Manager

```java
package com.example.nearneed;

import java.util.HashMap;
import java.util.Map;

/**
 * In-memory frontend state manager for booking statuses.
 * Acts as a single source of truth for the current session.
 *
 * Note: This is LOCAL state. For persistent state, see Firestore models.
 * This speeds up UI updates while Firestore persists data.
 *
 * Design Pattern: Singleton
 * - Only one instance exists throughout app lifecycle
 * - Accessed via getInstance()
 */
public class BookingStateManager {

    // Define all possible booking states as constants
    public static final String STATUS_ONGOING    = "ongoing";
    public static final String STATUS_COMPLETED  = "completed";
    public static final String STATUS_CANCELLED  = "cancelled";
    public static final String STATUS_PENDING    = "pending";

    // Singleton instance
    private static BookingStateManager instance;

    // In-memory storage: bookingId -> status
    private final Map<String, String> bookingStatuses = new HashMap<>();

    /**
     * Private constructor (singleton pattern)
     * Prevents multiple instances
     */
    private BookingStateManager() {}

    /**
     * Get or create singleton instance
     *
     * Lazy initialization: instance only created when first called
     */
    public static BookingStateManager getInstance() {
        if (instance == null) {
            instance = new BookingStateManager();
        }
        return instance;
    }

    /**
     * Store or update the status of a booking
     *
     * @param bookingId  Unique booking identifier
     * @param status     New status (use constants: STATUS_ONGOING, etc)
     */
    public void setStatus(String bookingId, String status) {
        bookingStatuses.put(bookingId, status);
    }

    /**
     * Get the current status of a booking
     *
     * @param bookingId  Booking to check
     * @return           Current status, or STATUS_ONGOING if not found
     */
    public String getStatus(String bookingId) {
        return bookingStatuses.getOrDefault(bookingId, STATUS_ONGOING);
    }

    /**
     * Convenience method - check if booking is cancelled
     */
    public boolean isCancelled(String bookingId) {
        return STATUS_CANCELLED.equals(bookingStatuses.get(bookingId));
    }

    /**
     * Convenience method - check if booking is completed
     */
    public boolean isCompleted(String bookingId) {
        return STATUS_COMPLETED.equals(bookingStatuses.get(bookingId));
    }

    /**
     * Example of validation - only allow certain transitions
     *
     * Valid transitions:
     * PENDING -> ONGOING (user accepts)
     * ONGOING -> COMPLETED (user completes)
     * PENDING or ONGOING -> CANCELLED (user cancels)
     */
    public boolean canTransition(String bookingId, String newStatus) {
        String currentStatus = getStatus(bookingId);

        // COMPLETED is a final state, can't change from it
        if (STATUS_COMPLETED.equals(currentStatus)) {
            return false;
        }

        // Can't go from CANCELLED back to other states
        if (STATUS_CANCELLED.equals(currentStatus)) {
            return false;
        }

        return true;  // Otherwise allow transition
    }
}
```

**Key Concepts Explained:**

- **Singleton Pattern**: `getInstance()` ensures only one instance exists
- **getOrDefault()**: Returns value if exists, else default (safer than direct get)
- **State Validation**: `canTransition()` enforces allowed transitions
- **In-Memory vs Firestore**: This is LOCAL state for speed; Firestore persists actual data

## 5. Booking Status Updates with Conflict Handling

### Brief Explanation

**Status updates** handle transitions between states while **preventing conflicts** (e.g., both users trying to complete simultaneously). Uses version-based optimistic locking.

### Implementation Flow

```
User initiates status change (Complete, Cancel, etc)
    ↓
Validate transition is allowed
    ↓
Firestore transaction executes atomically
    ↓
Only one succeeds; others get error
    ↓
UI shows result or conflict message
```

### Conflict Scenarios & Handling

```
Scenario 1: Simultaneous Completion
- Seeker presses "Complete" at 10:00:01
- Provider presses "Complete" at 10:00:02
→ Solution: Version check - only first succeeds

Scenario 2: Network Race Condition
- User 1 makes change, goes offline
- User 2 makes different change
- User 1 comes online, tries to apply old change
→ Solution: Server validates version before applying

Scenario 3: Invalid Transition
- User tries to complete CANCELLED booking
→ Solution: State validation rejects transition
```

### Edge Cases / Error Handling

```java
// Handle invalid transitions
if (!isValidTransition(currentState, newState)) {
    showError("Cannot " + newState + " from " + currentState);
    return;
}

// Handle network errors
onError(IOException e) {
    showError("No internet - will retry when online");
    queueForRetry(updateRequest);
}

// Handle concurrent modifications
if (versionMismatch) {
    showMessage("Status changed by another user. Refreshing...");
    fetchLatestState();
}
```

#### **NotificationCenter.java** - Notification Management

```java
package com.example.nearneed;

import com.google.firebase.auth.FirebaseAuth;
import com.google.firebase.auth.FirebaseUser;
import com.google.firebase.firestore.CollectionReference;
import com.google.firebase.firestore.DocumentSnapshot;
import com.google.firebase.firestore.FirebaseFirestore;
import com.google.firebase.firestore.ListenerRegistration;
import com.google.firebase.firestore.Query;
import com.google.firebase.firestore.WriteBatch;

import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import java.util.UUID;

/**
 * NotificationCenter manages all notifications
 *
 * Features:
 * - Send notifications to users
 * - Fetch all notifications
 * - Listen to unread count in real-time
 * - Mark as read
 * - Delete notifications
 */
public final class NotificationCenter {

    private NotificationCenter() {}  // Utility class, no instances

    /**
     * Callback interface for when notifications load
     */
    public interface OnNotificationsLoaded {
        void onLoaded(List<AppNotification> notifications);
    }

    /**
     * Callback interface for unread count changes
     */
    public interface OnCountChanged {
        void onChange(int count);
    }

    /**
     * Send notification to currently logged in user
     */
    public static void addNotification(String title, String message) {
        FirebaseUser user = FirebaseAuth.getInstance().getCurrentUser();
        if (user == null) return;
        sendNotificationToUser(user.getUid(), title, message);
    }

    /**
     * Send notification to a specific user by ID
     *
     * Process:
     * 1. Get reference to user's notifications subcollection
     * 2. Create notification document with all details
     * 3. Set read = false (unread)
     * 4. Upload to Firestore
     *
     * Firestore path: users/{userId}/notifications/{notificationId}
     */
    public static void sendNotificationToUser(String userId, String title, String message) {
        if (userId == null || userId.isEmpty()) return;

        FirebaseFirestore db = FirebaseFirestore.getInstance();
        CollectionReference ref = db.collection("users")
                                     .document(userId)
                                     .collection("notifications");

        // Generate unique ID
        String id = UUID.randomUUID().toString();

        // Create notification data
        Map<String, Object> data = new HashMap<>();
        data.put("id", id);
        data.put("title", title);
        data.put("message", message);
        data.put("timestamp", System.currentTimeMillis());
        data.put("read", false);  // Mark as unread

        // Upload to Firestore
        ref.document(id).set(data);
    }

    /**
     * Fetch all notifications for current user (one-time fetch)
     * Used when opening the notification center popup
     *
     * Gets newest notifications first, limited to 50
     */
    public static void fetchOnce(OnNotificationsLoaded callback) {
        CollectionReference ref = notifRef();
        if (ref == null) {
            callback.onLoaded(new ArrayList<>());
            return;
        }

        // Query: order by timestamp DESC, limit 50
        ref.orderBy("timestamp", Query.Direction.DESCENDING)
                .limit(50)
                .get()
                .addOnSuccessListener(snap -> {
                    List<AppNotification> list = new ArrayList<>();
                    for (DocumentSnapshot doc : snap.getDocuments()) {
                        AppNotification notif = fromDoc(doc);
                        if (notif != null) {
                            list.add(notif);
                        }
                    }
                    callback.onLoaded(list);
                })
                .addOnFailureListener(e -> callback.onLoaded(new ArrayList<>()));
    }

    /**
     * Attach real-time listener for unread count
     * Calls callback whenever count changes
     *
     * Returns registration to stop listening later
     */
    public static ListenerRegistration listenUnreadCount(OnCountChanged callback) {
        CollectionReference ref = notifRef();
        if (ref == null) {
            callback.onChange(0);
            return null;
        }

        return ref.whereEqualTo("read", false)
                .addSnapshotListener((snapshot, error) -> {
                    if (error != null) {
                        callback.onChange(0);
                        return;
                    }

                    int count = 0;
                    if (snapshot != null) {
                        count = snapshot.size();  // Number of unread docs
                    }
                    callback.onChange(count);
                });
    }

    /**
     * Mark a notification as read
     */
    public static void markAsRead(String notificationId) {
        CollectionReference ref = notifRef();
        if (ref == null) return;

        ref.document(notificationId).update("read", true);
    }

    /**
     * Delete a notification
     */
    public static void deleteNotification(String notificationId) {
        CollectionReference ref = notifRef();
        if (ref == null) return;

        ref.document(notificationId).delete();
    }

    /**
     * Get reference to current user's notifications subcollection
     * Returns null if user not logged in
     */
    private static CollectionReference notifRef() {
        FirebaseUser user = FirebaseAuth.getInstance().getCurrentUser();
        if (user == null) return null;

        return FirebaseFirestore.getInstance()
                .collection("users")
                .document(user.getUid())
                .collection("notifications");
    }

    /**
     * Convert Firestore document to AppNotification object
     */
    private static AppNotification fromDoc(DocumentSnapshot doc) {
        if (!doc.exists()) return null;

        return new AppNotification(
            doc.getString("id"),
            doc.getString("title"),
            doc.getString("message"),
            doc.getLong("timestamp") != null ? doc.getLong("timestamp") : 0L,
            doc.getBoolean("read") != null ? doc.getBoolean("read") : false
        );
    }
}
```

**Key Functions Explained:**

- `addSnapshotListener()`: Real-time listener for unread count changes
- `whereEqualTo("read", false)`: Query filter for unread notifications only
- `orderBy(...DESCENDING)`: Newest notifications first
- `UUID.randomUUID()`: Generate unique ID for each notification

### Firebase Structure

```
Firestore:
├── users/
│   └── {userId}
│       └── notifications/ (subcollection)
│           └── {notificationId}
│               ├── id: "abc-123"
│               ├── title: "New message from Sarah"
│               ├── message: "Hey, are you available tomorrow?"
│               ├── timestamp: 1682774400000
│               ├── read: false
│               └── type: "message" (optional for filtering)
```
